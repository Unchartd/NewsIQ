# NewsIQ — AI News Intelligence Platform

* **Version**: 1.0 (Production Baseline)
* **Status**: Approved & Frozen
* **Last Updated**: 2026-07-10

> Understand any major story across the globe in under 30 seconds, with transparency, neutrality, and trust.

NewsIQ is a dense, multi-source AI comprehension engine built for professionals who value speed, accuracy, and depth over endless scrolling. By aggregating data from numerous sources, NewsIQ automatically clusters events, extracts factual timelines, and identifies narrative differences between publishers—delivering a singular, trustworthy narrative.

---

## ⚡ Features & Capabilities

- **AI Summarization (Pipeline C):** Get the headline, a one-line summary, and an executive brief—all generated by capability-matched LLMs.
- **Difference Engine:** Compare how different sources are reporting the same event. Automatically detect missing facts, contradictions, and focus areas.
- **Fact Extraction:** Instantly view structured data like key locations, people, dates, and organizations involved in a story.
- **Trend Engine:** Discover what the world is talking about. Stories are algorithmically ranked based on recency, publisher volume, and real-time user engagement.
- **Personalized Feed:** Logged-in users receive a tailored news feed based on their reading history and explicitly chosen categories.
- **Account Security:** Includes a complete user authentication flow featuring email verification and secure forgot-password flows.
- **Premium Broadstreet UI:** A visually stunning, high-density reader interface styled like a modern broadsheet, ensuring zero visual noise.

## 🏗️ Architecture & Tech Stack

NewsIQ is built as a modular monorepo, splitting concerns between Next.js frontends and an AI-native Python backend.

### 🌐 System Pipeline Overview

```
[Pipeline A: Discovery] ➔ [Pipeline B: Event Intelligence] ➔ [Pipeline C: Story Synthesis]
```

1. **Pipeline A (Discovery & Ingestion)**: Manages RSS feeds crawling, HTML extraction, bloom filter deduplication, validation, and HDBSCAN event clustering.
2. **Pipeline B (Event Intelligence)**: Evaluates event lifecycle stages, resolves canonical event identities, and links cross-article entities.
3. **Pipeline C (Story Synthesis)**: Runs a 7-stage orchestrator (Knowledge Graph, Contradiction, Source Comparison, Timeline Compiler, Summary Generator, Feedback QA, and Publisher) storing immutable artifact snapshots.

### Frontend App Ecosystem
- **`apps/web`**: The main user-facing broadsheet portal. Next.js 16 (App Router), React 19, TailwindCSS v4, Zustand.
- **`apps/admin`**: Administrative dashboard. Displays real-time ingestion throughput, story distribution, provider health, latency metrics, and version rollback triggers. Next.js 16, React 19, Recharts.

### Backend App Ecosystem (`apps/api`)
- **Framework:** FastAPI (Python)
- **Database:** PostgreSQL (Primary), Redis (Caching/Budgets), Qdrant (Vector Search)
- **Message Broker:** Celery / Redis
- **Observability:** Sentry SDK, Prometheus Metrics, Structured JSON Logging

## 🚀 Quick Start & Development Setup

### Prerequisites
- [Docker](https://www.docker.com/) & Docker Compose
- Node.js 20+ & npm
- Python 3.11+ & `uv` (recommended)

### 1. Setup Environment
Clone the repository and set up your local environment variables:
```bash
cp .env.example .env
```

---

### Option A: Running with Docker Compose (Recommended)
This boots the entire stack—including PostgreSQL, Redis, Qdrant, Meilisearch, the FastAPI backend, Celery workers, and both Next.js frontends.

```bash
# Start all services and build images
docker-compose up -d --build

# Seed the database with default news sources and categories
docker-compose exec api python -m app.scripts.seed
```

- **Frontend (Reader App):** `http://localhost:3000`
- **Admin Dashboard Console:** `http://localhost:3002`
- **Backend API Docs (Swagger):** `http://localhost:8000/docs`

---

### Option B: Running Local Servers (Native Setup)
For active development with step-through debugging, you can run the FastAPI backend and Next.js frontend locally while running only the database/caching infrastructure in Docker.

#### Step 1: Start Infrastructure (PostgreSQL, Redis, Qdrant, Meilisearch)
```bash
# Spin up only the backing databases and caches
docker-compose up -d postgres redis qdrant meilisearch
```

#### Step 2: Run Backend API Locally (`apps/api`)
```bash
cd apps/api

# Create and activate virtual environment (using uv for speed)
uv venv
.venv\Scripts\activate      # On Windows
# source .venv/bin/activate  # On macOS/Linux

# Install dependencies and package
uv pip install -e ".[dev]"

# Run database migrations
uv run alembic upgrade head

# Seed initial database records
uv run python -m app.scripts.seed

# Start the uvicorn development server
uv run uvicorn app.main:app --host 127.0.0.1 --port 8000 --reload
```

#### Step 3: Run Frontend Reader Locally (`apps/web`)
```bash
cd apps/web
npm install
npm run dev
```

#### Step 4: Run Admin Dashboard Locally (`apps/admin`)
```bash
cd apps/admin
npm install
npm run dev
```
