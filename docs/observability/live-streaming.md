# Real-Time Event & Log Streaming

This document details the real-time event streaming system implemented using Server-Sent Events (SSE).

## System Architecture

We utilize Server-Sent Events (SSE) instead of WebSockets because pipeline telemetry is unidirectional (server-to-client). SSE runs over HTTP/1.1 or HTTP/2, uses standard text formatting, and automatically manages connection drops and retries.

```
[Celery Workers / StageSpans]
            │
            ▼ (Redis PUBLISH)
   [Redis Pub/Sub Bus]
            │
            ▼ (SSE Subscription)
    [FastAPI Endpoints] ──(HTTP Streaming Response)──> [React Frontend]
```

## SSE Endpoints

### 1. Pipeline Run Status Stream
*   **Path**: `/api/v1/admin/pipeline/stream`
*   **Description**: Streams stage execution state transitions (Pending -> Running -> Success/Failed) for the active pipeline run.
*   **Redis Channel**: `newsiq-pipeline-events`
*   **Payload Format**:
    ```json
    {
      "run_id": "908fd64b-...",
      "trace_id": "a984df6d-...",
      "stage": "ingestion_rss",
      "status": "running",
      "started_at": "2026-06-21T20:30:00Z"
    }
    ```

### 2. Live Log Stream
*   **Path**: `/api/v1/admin/pipeline/runs/{run_id}/stages/{stage}/logs/stream`
*   **Description**: Streams stdout/stderr logging generated by a specific running stage.
*   **Redis Channel**: `newsiq:logs:{run_id}:{stage}:stream`
*   **Behavior**: When a client connects, the endpoint first reads all existing logs from the Redis list at key `newsiq:logs:{run_id}:{stage}` to populate the terminal log history, then subscribes to the Pub/Sub channel to yield incoming logs in real time.
